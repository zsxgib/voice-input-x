# 语音输入法设计方案

## 目标

将 `video_to_text.py` 转换为 Linux 平台的语音输入法工具，按键触发录音，识别后预览确认，再输入到当前应用。

## 使用场景

- 用户在任意应用（浏览器、终端、编辑器等）中需要语音输入
- 按下全局热键开始录音，再次按下结束录音
- 预览识别结果，确认后自动粘贴到当前光标位置

## 整体架构

```
┌─────────────────┐
│ Ctrl+Shift+V   │  ← 全局热键启动/停止
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 录音模块        │  ← 显示悬浮窗 + 计时
│ (最长 5 分钟)  │  ← 按 Enter 结束录音
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ Whisper 识别   │  ← small 模型 (<1秒)
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 预览窗口       │  ← Enter 确认，Esc 取消
│ [确认] [取消]  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│ 文字插入       │  ← xdotool 模拟输入
└─────────────────┘
```

## 功能需求

### 1. 全局热键
- 默认热键：`Ctrl+Shift+V`
- 支持自定义配置
- 启动/停止录音复用同一热键

### 2. 录音
- 最大时长：5 分钟
- 结束方式：按 Enter 键 或 超时自动结束
- 显示悬浮窗，包含：
  - 录音状态指示
  - 计时器
  - 结束按钮

### 3. 语音识别
- 模型：Faster-Whisper `small`（平衡速度和准确率）
- 语言：中文（可扩展支持多语言）
- 目标延迟：< 1 秒

### 4. 预览窗口
- 显示识别结果文本
- 确认按钮：Enter 或 点击
- 取消按钮：Esc 或 点击

### 5. 文字插入
- 方式：模拟 Ctrl+V 粘贴
- 备选：直接用 xdotool type 输入

## 技术选型

| 模块 | 方案 | 说明 |
|------|------|------|
| 全局热键 | python-keybinder3 | 跨桌面环境支持 |
| 录音 | sounddevice | 简单易用 |
| 语音识别 | Faster-Whisper | 复用现有方案 |
| 预览窗口 | tkinter | 标准库，简单 |
| 文字插入 | xdotool | Linux 通用 |

## 性能目标

| 指标 | 目标 |
|------|------|
| 启动延迟 | < 200ms |
| 识别延迟 | < 1 秒 (5分钟音频) |
| 总流程 | < 3 秒 |

## 配置项

```python
# 可配置项
HOTKEY = "Ctrl+Shift+V"      # 全局热键
MAX_DURATION = 300           # 最大录音时长（秒）
WHISPER_MODEL = "small"      # Whisper 模型
LANGUAGE = "zh"              # 识别语言
OLLAMA_URL = "http://localhost:11434/api/generate"
```
